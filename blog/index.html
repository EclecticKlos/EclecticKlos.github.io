Ha-rrays! and ashes

Potential brief summary of the article?

So, you want to know about arrays and hashes, eh? For an array, imagine you need to make a step-by-step check list for someone to check their email who had never checked their email before. First, start computer. Second, login as computer user. Third, open web browser. Etc etc. Arrays are essentially lists, where each item on the list is assigned a numbered slot. This is great when you have data that is unconnected to any single other piece of data. But what if the information is paired, for instance with a list of screen names to users?

A hash table would be perfect for this job. Hashes are basically an array, but instead of each element being paired with a number, it is paired with another piece of data, together they are known as a key-value pair. So for example, if GitHub were to store my screen name in a hash, in the key-value pair "Dan Klos" would be the key, and EclecticKlos would be the value.
